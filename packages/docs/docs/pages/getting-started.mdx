# Quick Start Guide

Get **Claude Swarm** up and running in your development environment in just a few minutes.

:::warning[Prerequisites]
Before you begin, make sure you have:
- **Node.js** 18+ or **Bun** 1.0+
- **Git** 2.25+ (for worktree support)
- **tmux** 3.0+ (optional, for session management)
:::

## Installation

### Option 1: Using npm

```bash
# Install globally
npm install -g @claude-swarm/cli

# Verify installation
claude-swarm --version
```

### Option 2: Using Bun (Recommended)

```bash
# Install globally with Bun
bun add -g @claude-swarm/cli

# Verify installation
claude-swarm --version
```

### Option 3: Using npx (No installation)

```bash
# Run directly without installing
npx @claude-swarm/cli --help
```

## Initial Setup

### 1. Configure Your Environment

```bash
# Initialize configuration
claude-swarm init

# This creates ~/.claude-swarm/config.json with defaults
```

### 2. Set Up GitHub Integration (Optional)

```bash
# Add GitHub token for issue integration
claude-swarm config set github.token YOUR_GITHUB_TOKEN

# Configure default repository
claude-swarm config set github.defaultRepo owner/repository
```

### 3. Configure Tmux (Optional)

```bash
# Enable tmux integration
claude-swarm config set tmux.enabled true

# Set session prefix
claude-swarm config set tmux.sessionPrefix "cs-"
```

## Your First Workflow

### Setting Up a Project

```bash
# Navigate to your project directory
cd /path/to/your/project

# Set up Claude Swarm for this project
claude-swarm setup my-awesome-project
```

This command:
- ‚úì Validates the git repository
- ‚úì Creates the base worktree structure
- ‚úì Initializes tmux session management
- ‚úì Sets up agent isolation directories

### Working on a Task

```bash
# Start working on a specific GitHub issue
claude-swarm work-on-task --issue 123

# Or work on a custom branch
claude-swarm work-on-task --branch feature/new-login

# Or work with a specific agent ID
claude-swarm work-on-task --issue 123 --agent-id login-specialist
```

This creates:
- üå≥ **Isolated worktree** for the task
- üñ•Ô∏è **Dedicated tmux session** (if enabled)
- üìã **Agent-specific context** directory
- üîó **GitHub issue linkage** (if applicable)

### Code Examples with Syntax Highlighting

```typescript {4,8-12} showLineNumbers filename="src/core/worktree.ts"
import { GitOps } from './git'
import { ErrorFactory, ERROR_CODES } from '../shared/errors'

export class WorktreeManager {
  constructor(private gitOps: GitOps) {}

  async createWorktree(options: WorktreeOptions): Promise<Worktree> {
    // Validate repository state
    const isValid = await this.gitOps.isValidRepository(options.basePath)
    if (!isValid) {
      throw ErrorFactory.worktree(ERROR_CODES.WORKTREE_INVALID_REPOSITORY)
    }

    // Create the worktree
    return await this.gitOps.createWorktree({
      path: options.path,
      branch: options.branch,
      upstream: options.upstream
    })
  }
}
```

### Managing Your Workspace

```bash
# List all active worktrees
claude-swarm list

# Switch to a different worktree/session
claude-swarm switch feature-123

# Check status of current workspace
claude-swarm status
```

### Cleaning Up

```bash
# Clean up a specific worktree
claude-swarm cleanup --worktree feature-123

# Clean up all inactive worktrees
claude-swarm cleanup --all

# Clean up and preserve important files
claude-swarm cleanup --worktree feature-123 --preserve-context
```

## Common Workflows

### üêõ Bug Fix Workflow

```bash
# 1. Start working on bug report
claude-swarm work-on-task --issue 456 --agent-id bug-hunter

# 2. Your environment is ready!
# - Worktree: /project/worktrees/bug-456
# - Tmux session: cs-bug-456
# - Context: ~/.claude-swarm/contexts/bug-hunter

# 3. After fixing, clean up
claude-swarm cleanup --worktree bug-456
```

### ‚ú® Feature Development

```bash
# 1. Start feature work
claude-swarm work-on-task --branch feature/user-auth --agent-id auth-expert

# 2. Develop in isolation
# 3. When ready, cleanup preserves your work
claude-swarm cleanup --worktree feature-user-auth --preserve-context
```

### üìä Code Review Preparation

```bash
# 1. Set up review environment
claude-swarm work-on-task --branch review/pr-789 --agent-id reviewer

# 2. Review changes
# 3. Clean up review workspace
claude-swarm cleanup --worktree review-pr-789
```

:::tip[Pro Tips]
- Use **descriptive agent IDs** to organize different types of work
- The `--preserve-context` flag keeps important files when cleaning up
- Set up **shell aliases** for common commands:
  ```bash
  alias csw='claude-swarm work-on-task'
  alias csc='claude-swarm cleanup'
  alias csl='claude-swarm list'
  ```
:::

## Troubleshooting

### Common Issues

#### Git Worktree Errors
```bash
# If you get "worktree already exists" errors
claude-swarm cleanup --worktree WORKTREE_NAME --force

# Check git worktree status
git worktree list
```

#### Tmux Session Issues
```bash
# List all tmux sessions
tmux list-sessions

# Kill stuck session
tmux kill-session -t cs-SESSION_NAME
```

#### Configuration Issues
```bash
# View current configuration
claude-swarm config list

# Reset to defaults
claude-swarm config reset
```
